# Docs-as-Code Training
This repo is reserved for Technical Publications - Structured Upskilling Program.
## Overview
Doc-as-Code stack
1. Markdown (writing layer)
2. GitHub (versioning and collaboration)
3. Static site generators (publishing layer)
4. CI/CD basics (automation layer)
5. Docs workflow (SME reviews, branching, etc.)

Docs are written in Markdown, stored in Git, reviewed via pull requests, and published via automation pipelines.
